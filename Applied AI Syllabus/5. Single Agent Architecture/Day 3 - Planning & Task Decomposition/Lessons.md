## Part C — Experiment: no planning vs. plan-first vs. dynamic replanning
Per the syllabus showcase task: give the agent a complex 5-step task and compare three run modes:
1. No planning — each step decided from scratch, same as a plain Week 3 ReAct loop.
2. Explicit plan-first — a full ordered subtask list with dependencies, generated by one upfront LLM call, executed in order.
3. Dynamic replanning — plan-first, but with a replan trigger wired in that re-runs the planning call with the plan-so-far plus whatever real tool result broke an assumption (a broken assumption mid-execution regenerates the remaining plan).

Watch specifically for: does (1) lose the overall thread or redo work compared to (2)/(3)? Does (2) get stuck or produce a wrong final result when reality diverges from an early assumption, in a way (3) recovers from? Is (3)'s extra replanning call cost visibly justified by the outcome difference, or does it only pay off when something actually breaks mid-run?
